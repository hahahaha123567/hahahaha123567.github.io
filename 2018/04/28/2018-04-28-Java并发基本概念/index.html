    Java的多线程

    几个常识

    从JDK1.2之后，Java的线程模型就由OS的原生线程模型实现

    Java使用抢占式调度多线程系统，由系统分配执行时间

    设置线程的优先级不是很靠谱，因为有的OS线程模型本身的优先级不够多

    线程状态

    1. New
    2. Runnable: running / ready
    3. Waiting: 无限等待直到被其他线程显式唤醒
    4. Timed waiting: 等待被系统自动唤醒
    5. Blocked: 等其他线程解锁
    6. Terminated

    状态转换

    Object.wait() -> Waiting

    Object.wait(time) -> Timed waiting

    Object.notify() -> Runnable

    Thread.join() -> Waiting

    Thread.join(time) -> Timed waiting

    Thread.sleep() -> Timed waiting

    Java Memory Model

    内存模型

    1. main memory
    2. Working memory
    3. Java thread

    交互操作

    以下操作可以认为是满足原子性的

    1. lock
    2. unlock
    3. read
    4. load
    5. use
    6. assign
    7. store
    8. write

    原子性、可见性、有序性

    Atomicity

    synchronized 调用字节码指令 monitorenter 和 monitorexit，使用JMM的lock, unlock实现

    Visibility

    通过主内存传输，volatile保证新值能立即同步到主内存

    synchronized 和 final 也可以实现

    Ordering

    线程内有序，线程间无序（指令重排序，工作内存和主内存同步有延迟）

    volatile 禁止指令重排序

    synchronized 也可以实现

    Happens-before

    先行发生原则

    逻辑顺序，不是时间顺序
